# AI Agent 使用規則

AI agent 可以協助撰寫程式、文件、測試與除錯，但需遵守以下規則。

1. 每次修改前，先確認目前任務目標與影響範圍。
2. 不要任意刪除或重寫組員已完成的檔案。
3. 不要新增超出 MVP 範圍的大型功能。
4. 不要修改 API 行為，除非已同步更新文件並通知組員。
5. 修改 mock data 時需維持 JSON 格式正確。
6. 分支完成回報需標註是否有使用 AI agent 協助。
7. 每次完成後需留下測試方式或驗證結果。

## 建議提示詞

```text
任務名稱：
- 請填入任務名稱

預期修改檔案：
- 請填入預期修改檔案

驗收條件：
1. 請填入驗收條件

完成後請說明修改了哪些檔案、如何測試，以及是否有需要人工確認的地方。
```
